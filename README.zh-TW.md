# Delphi Agent Skill

[English](README.md) | [正體中文 (zh-TW)](README.zh-TW.md) | [简体中文 (zh-CN)](README.zh-CN.md)

**Delphi Agent** 是一個 AI agent 技能，使用「德爾菲法」（Delphi Technique），幫助使用者透過結構化、多回合的專家共識來解決複雜問題。

## 🎯 目標與理念

許多現實世界中的決策都牽涉到真正的權衡取捨，並沒有單一的正確答案。Delphi Agent 將結構化專家辯論的力量帶入單一聊天視窗中——不需要部署多智能體平台。該 agent 模擬了一名協調者 (Coordinator) 和 2–5 名持有相反價值立場的虛擬專家，引導他們經歷盲測定位、交叉詰問、紅線宣告和強制收斂，然後提交一份包含具體建議的統一最終報告 (Final Report)。

## ⚙️ 運作方式

Delphi Agent 遵循一個分為 4 個階段的結構化流程：

| 階段 | 名稱 | 內容 |
|-------|------|-------------|
| **第 1 階段** | 初始化 (Initialization) | 協調者解構問題，提出專家典範和配置，並等待您的確認。 |
| **第 2 階段** | 辯論回合 (Debate Rounds) | 專家在多個回合中進行辯論——互相提出挑戰，同時逐漸鎖定子議題的共識。在紅線回合之前都可以引入新的變數。 |
| **第 3 階段** | 強制收斂 (Forced Convergence) | 必須解決核心困境。專家們必須妥協，或者提出正式的異議 (Formal Dissent)。 |
| **第 4 階段** | 最終報告 (Final Report) | 一份獨立的報告，包含執行摘要、收斂路徑、行動計畫、權衡地圖以及殘留風險。 |

## 🚀 使用案例與範例問題

Delphi Agent 專為那些沒有明顯「正確答案」且需要跨領域思考的狀況而設計。以下每個例子都是可以直接使用的範例問題——只需複製並貼上即可開始。

### 1. 策略決策
當面臨重大轉型、架構選擇或商業策略，而不同的價值觀或優先級產生衝突時。

> *"我們正在為新的 MVP 評估微服務 (microservices) 和單體架構 (monolith)。請建立一個包含「交付速度最大化者」、「系統可靠性純粹主義者」與「技術債務最小化者」的 Delphi 小組。"*

### 2. 跨領域問題解決
當一個挑戰需要綜合那些極少能達到一致的衝突約束時。

> *"為我們新的 AI 工具設計一份資料隱私政策。讓一名「法規遵循捍衛者」、「無約束資料創新者」和「使用者主權倡議者」來辯論其中的權衡取捨。"*

### 3. 盲點檢測與壓力測試
當您的團隊傾向於某個想法，但您希望能透過根本對立的典範對其進行嚴格的壓力測試，以發現隱藏的盲點時。

> *"我的團隊想將我們整個資料庫遷移到 MongoDB。請扮演三種完全對立的資料架構典範，找出這個計畫中的缺陷。"*

### 4. 預測與風險評估
當試圖在高度不確定且最佳化條件互相衝突的環境中，預測未來趨勢或評估風險時。

> *"未來 10 年內，量子運算將如何影響目前的區塊鏈密碼學？將一名「去中心化理想主義者」、「後量子現實主義者」與「國家級威脅建立模型者」聚集在一起討論。"*

### 5. 道德困境分析
當一項決策牽涉到根本的價值觀衝突，沒有客觀正確的答案，而您需要透明地將權衡取捨呈現出來時。

> *"一輛自駕車煞車失靈。它必須做出選擇：(1) 維持路線以拯救乘客，或 (2) 轉向撞牆以拯救行人。人工智慧應該被設定為拯救誰？"*

## 💡 使用與觸發方式

### 觸發方式 (How to Use)

您可以透過兩種方式使用 Delphi Agent：
* **作為 Skill 安裝：** 參考 [`SKILL.md`](./delphi-agent/SKILL.md) 檔案將其作為 agent skill 安裝。
* **作為 Gemini Gem 使用：** 參考 [`gemini-gem.md`](./gemini-gem.md) 了解格式正確的 Gemini Gem 設定。

### 如何觸發 (How to Trigger)

Delphi Agent 會對廣泛的提示做出反應：
* **明確的關鍵字：** "Delphi", "expert consensus", "structured debate", "devil's advocate", "pros and cons"
* **決策模式：** 權衡取捨、比較替代方案 ("A vs B", "should I X or Y")、策略或架構選擇
* **猶豫不決的訊號：** "I'm stuck", "I can't decide", "help me think through this"

只需在您的提示詞中包含以上任意內容，agent 就會自動切換為協調者 (Coordinator) 角色，並開始設定流程。
