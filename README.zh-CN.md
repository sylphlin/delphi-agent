# Delphi Agent Skill

[English](README.md) | [正體中文 (zh-TW)](README.zh-TW.md) | [简体中文 (zh-CN)](README.zh-CN.md)

**Delphi Agent** 是一个 AI agent 技能，使用“德尔菲法”（Delphi Technique），帮助用户通过结构化、多回合的专家共识来解决复杂问题。

## 🎯 目标与理念

许多现实世界中的决策都牵涉到真正的权衡取舍，并没有单一的正确答案。Delphi Agent 将结构化专家辩论的力量带入单一聊天窗口中——不需要部署多智能体平台。该 agent 模拟了一名协调者 (Coordinator) 和 2–5 名持有相反价值立场的虚拟专家，引导他们经历盲测定位、交叉诘问、红线宣告和强制收敛，然后提交一份包含具体建议的统一最终报告 (Final Report)。

## ⚙️ 运作方式

Delphi Agent 遵循一个分为 4 个阶段的结构化流程：

| 阶段 | 名称 | 内容 |
|-------|------|-------------|
| **第 1 阶段** | 初始化 (Initialization) | 协调者解构问题，提出专家典范和配置，并等待您的确认。 |
| **第 2 阶段** | 辩论回合 (Debate Rounds) | 专家在多个回合中进行辩论——互相提出挑战，同时逐渐锁定子议题的共识。在红线回合之前都可以引入新的变量。 |
| **第 3 阶段** | 强制收敛 (Forced Convergence) | 必须解决核心困境。专家们必须妥协，或者提出正式的异议 (Formal Dissent)。 |
| **第 4 阶段** | 最终报告 (Final Report) | 一份独立的报告，包含执行摘要、收敛路径、行动计划、权衡地图以及残留风险。 |

## 🚀 使用案例与示例问题

Delphi Agent 专为那些没有明显“正确答案”且需要跨领域思考的状况而设计。以下每个例子都是可以直接使用的示例问题——只需复制并粘贴即可开始。

### 1. 战略决策
当面临重大转型、架构选择或商业战略，而不同的价值观或优先级产生冲突时。

> *"我们正在为新的 MVP 评估微服务 (microservices) 和单体架构 (monolith)。请建立一个包含‘交付速度最大化者’、‘系统可靠性纯粹主义者’与‘技术债务最小化者’的 Delphi 小组。"*

### 2. 跨领域问题解决
当一个挑战需要综合那些极少能达到一致的冲突约束时。

> *"为我们新的 AI 工具设计一份数据隐私政策。让一名‘合规捍卫者’、‘无约束数据创新者’和‘用户主权倡议者’来辩论其中的权衡取舍。"*

### 3. 盲点检测与压力测试
当您的团队倾向于某个想法，但您希望能通过根本对立的典范对其进行严格的压力测试，以发现隐藏的盲点时。

> *"我的团队想将我们整个数据库迁移到 MongoDB。请扮演三种完全对立的数据架构典范，找出这个计划中的缺陷。"*

### 4. 预测与风险评估
当试图在高度不确定且优化条件互相冲突的环境中，预测未来趋势或评估风险时。

> *"未来 10 年内，量子计算将如何影响目前的区块链密码学？将一名‘去中心化理想主义者’、‘后量子现实主义者’与‘国家级威胁建模者’聚集在一起讨论。"*

### 5. 道德困境分析
当一项决策牵涉到根本的价值观冲突，没有客观正确的答案，而您需要透明地将权衡取舍呈现出来时。

> *"一辆自动驾驶汽车刹车失灵。它必须做出选择：(1) 维持路线以拯救乘客，或 (2) 转向撞墙以拯救行人。人工智能应该被设定为拯救谁？"*

## 💡 使用与触发方式

### 如何使用 (How to Use)

您可以通过两种方式使用 Delphi Agent：
* **作为 Skill 安装：** 参考 [`SKILL.md`](./delphi-agent/SKILL.md) 文件将其作为 agent skill 安装。
* **作为 Gemini Gem 使用：** 参考 [`gemini-gem.md`](./gemini-gem.md) 了解格式正确的 Gemini Gem 设置。

### 如何触发 (How to Trigger)

Delphi Agent 会对广泛的提示做出反应：
* **明确的关键字：** "Delphi", "expert consensus" (专家共识), "structured debate" (结构化辩论), "devil's advocate" (魔鬼代言人), "pros and cons" (优缺点)
* **决策模式：** 权衡取舍、比较替代方案 ("A 与 B", "我应该选择 X 还是 Y")、战略或架构选择
* **犹豫不决的信号：** "我卡住了" (I'm stuck)、"我无法决定" (I can't decide)、"帮我想想这个问题" (help me think through this)

只需在您的提示词 (prompt) 中包含以上任意内容，agent 就会自动切换为协调者 (Coordinator) 角色，并开始设置流程。
